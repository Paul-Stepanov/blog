# Skill: Review

Инструкции для ревьюера кода.

---

## Обязанности

### 1. Качество кода
- Читаемость и понятность
- Правильное именование
- DRY, KISS, YAGNI
- Глубокая вложенность

### 2. Безопасность
- SQL-инъекции
- XSS, CSRF
- Раскрытие секретов
- Валидация входных данных

### 3. Производительность
- N+1 запросы
- Неэффективные алгоритмы
- Утечки памяти

### 4. Соответствие стандартам
- PSR-12
- Архитектура из Design
- Типизация

---

## Категории проблем

### 🔴 Critical (Блокирующие)
- Уязвимости безопасности
- Критические баги
- Нарушение архитектуры

**Действие:** ❌ Код НЕ может быть принят

### 🟡 Warning (Важные)
- Плохая производительность
- Дублирование кода
- Нарушение SOLID

**Действие:** ⚠️ Желательно исправить

### 🟢 Suggestion (Рекомендации)
- Рефакторинг для читаемости
- Дополнительная документация

**Действие:** 💡 Рассмотреть

---

## Чек-лист

### Типизация (PHP)
- [ ] `declare(strict_types=1)` присутствует
- [ ] Все аргументы типизированы
- [ ] Возвращаемые типы указаны
- [ ] Nullable типы `?Type`

### Безопасность
- [ ] Нет SQL-инъекций (prepared statements)
- [ ] Вывод экранируется
- [ ] Входные данные валидируются
- [ ] Нет захардкоженных секретов
- [ ] CSRF токены в формах

### Качество
- [ ] Нет дублирования (DRY)
- [ ] Функции < 20 строк
- [ ] Вложенность < 4 уровней
- [ ] Осмысленные имена

### Архитектура
- [ ] Соответствует Design
- [ ] Интерфейсы реализованы
- [ ] Паттерны применены корректно
