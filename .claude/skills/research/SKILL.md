# Skill: Research

Инструкции для исследователя кодовой базы.

---

## Обязанности

### 1. Анализ требований
- Понять что нужно реализовать
- Выявить неявные требования
- Определить граничные случаи
- Идентифицировать зависимости

### 2. Исследование кода
- Найти похожие реализации (Grep, Glob)
- Понять существующие паттерны
- Определить точки интеграции
- Выявить потенциальные конфликты

### 3. Поиск документации
- context7 для документации библиотек
- web-search для best practices
- Примеры в проекте

---

## Методология

### Шаг 1: Первичный анализ
1. Прочитать CLAUDE.md
2. Изучить структуру директорий (Glob)
3. Найти конфигурационные файлы
4. Понять стек технологий

### Шаг 2: Глубокое погружение
1. Найти код связанный с задачей (Grep)
2. Прочитать релевантные файлы (Read)
3. Понять зависимости
4. Изучить паттерны

### Шаг 3: Внешние источники
1. context7 для библиотек
2. web-search для best practices

### Шаг 4: Анализ через sequential-thinking
**ОБЯЗАТЕЛЬНО** использовать `mcp__sequential-thinking__sequentialthinking`:
- Минимум 5 шагов
- Продумать риски
- Сформулировать рекомендации
